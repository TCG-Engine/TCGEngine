# SOR — Complex / Remaining-Cards Implementation Plan

Final SOR implementation roadmap. Covers the **35 cards** still unimplemented after the Simple plan ([sor-simple-plan.md](sor-simple-plan.md)) and Medium plan ([sor-medium-plan.md](sor-medium-plan.md)) — both of which are *batch-complete*. Card text, stats, and per-card notes live in [sor-implement.md](sor-implement.md); this plan only sequences the work and tells you when to invoke the skill.

## ⚠ Why this plan exists (read first)

Two distinct groups are bundled here:

1. **~18 Simple/Medium-tier cards that were tier-classified but never batched.** They have **generated trigger stubs** (the generator detected a WhenPlayed/OnAttack/etc.) but **no Custom handler was ever written**, so the trigger fires and dispatches to nothing — the ability silently no-ops in the live game. Most reuse infra that already exists; they are *not* hard, they just slipped through the cracks. They go **first** (Phases A–D) to clear the easy backlog and de-risk.
2. **~14 genuinely-Complex cards** needing new infrastructure: cross-player decision routing, a "choose N of a list" modal decision, full-deck search, cross-arena attack, iterative reveal-until-stop, event-cost surcharge / event-blanking passives, and the aspect-penalty-ignore hook. These go **last** (Phases E–H).
3. **3 unbuilt leaders** (SOR_003, SOR_008, SOR_013) are slotted next to the mechanic they need.

Pure-keyword/vanilla cards (SOR_180 Shielded, SOR_195 Ambush, etc.) are auto-wired by the dictionaries and are **not** in scope.

## How to use this plan

- **Each batch = one `swusim-implement-card` invocation.** The skill looks up card text, writes **all DSL tests first**, then proceeds (no hard STOP for Simple/Medium — see [[feedback-implement-card-gate]]; STOP only on a Hard/design-fork card, flagged ⛔ below).
- **Per batch:** invoke skill → (review tests) → implement → run regression → **retro** → check off below.
- **End-of-batch retro:** after green, read `docs/swu-impl-retro.txt` and report lessons; fold approved card-impl lessons into the `swusim-implement-card` skill. (memory: [[feedback-batch-retro]])
- **Test runner (never host PHP):** `curl http://localhost:3400/TCGEngine/zzRegressionSWUSim.php` (currently **515 passing, 0 failed**).
- **Never commit.** The user commits manually.
- **Re-tier at batch time.** If the skill's Step-1 research shows a card needs more than its phase's mechanic, move it.

> **When ready to start a batch, invoke:** `swusim-implement-card` with the listed card IDs.

## Infrastructure already in place (verified — do NOT rebuild)

- **Universal target handlers:** `DEAL_UNIT_DAMAGE`, `DEFEAT_UNIT`, `EXHAUST_UNIT`, `READY_UNIT`, `BOUNCE_UNIT`, `GIVE_SHIELD`, `GIVE_EXPERIENCE|N`, `HEAL_TARGET|N`, `APPLY_PHASE_BUFF|p|h`, `APPLY_PHASE_DEBUFF|p|h`, `GRANT_PHASE_KEYWORD|KW`, `SPLIT_DAMAGE`, `DISCARD_FROM_OWN_HAND`, `DISCARD_FROM_OPP_HAND`, `MZMAYCHOOSE` (you-may), `MZMULTICHOOSE` (up-to-N), `OPTIONCHOOSE` (named branch).
- **Attack:** `BeginSWUAttack` (effect-driven attack with a chosen unit), `SWUAddAttackPowerBonus` (+N power this attack), per-attack TurnEffect durations (`@attack`, expired in `SWUExpireTurnEffects('attack')` at end of `SWUCombatDamage`).
- **Turn-effect registry** ([[swusim-turn-effects]]): CardID-token grants `CARDID[#params][@duration]` (durations `attack`/`phase`/`perm`); keyword/stat readers `SWUHasTurnEffectKeyword`/`SWUTurnEffectKeywordValue(Sum)`/`SWUTurnEffectStatBonus`. Granting a keyword for a phase/attack is one `AddTurnEffect`.
- **Combat triggers** (Phase 7 of the medium plan): `OnAttackEnd` (`$onAttackEndAbilities`), `WhenDealsCombatDamage`, `WhenDefeats`, reactive on-enemy-defeat. **`LostAbilities($obj)`** subsystem (units that lose all abilities).
- **Cost:** `SWUComputePlayCost($player,$obj,$host=null)` single source of truth; `$playCostModifiers`/`$playCostFieldModifiers` registries; `ActivateCard($p,$mz,$ignoreCost,$discount,$prepaid)`; `SWUBeginPlayCard` hand-play orchestrator; `SWUHandPlayablesAtDiscount`.
- **Enter-ready + temp units:** `$gForceEnterReady` (SOR_129), regroup-/round-start hooks (SOR_186 can't-ready, SOR_017/SOR_193 pending defeat).
- **Play-from-discard:** `SWUPlayFromDiscard` (with discount threading).
- **Phase/round flags:** `SWU_DMGBASE_{target}` (base damaged this phase — SOR_175), `SWU_CARDS_PLAYED`, `SWU_PLAYED_UNIT_{uid}`, `SWU_ATTACKED_{uid}`; pattern = set in the game function, check in handler, clear in `RegroupPhaseStart` via `SWUClearGlobalEffectsByPrefix`.
- **Cross-player decision precedent:** SOR_167 routes a discard decision to the *opponent* inline (1 card) vs a routed MZCHOOSE (2+). The opponent-decision phase generalizes this — mind the **relative-mzID `$playerID` gotcha** ([[swusim-project]] session-41 KEY ENGINE GOTCHA): a handler that queues a relative-mzID decision must leave `$playerID =` the decision's player or `MZCountChoices` auto-resolves to `-`.

**Genuinely NEW infra built in this plan:** a "choose N of a named list" modal decision + UI (Phase F); cross-player target-choice routing as a reusable helper (Phase E); full-deck search (Phase G); cross-arena attack targeting (Phase G); iterative reveal-until-stop (Phase G); event-cost surcharge + first-event-blank passives (Phase G); aspect-penalty-ignore hook (Phase H).
